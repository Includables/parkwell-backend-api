Park api:

This api handles all processing under the control of the park application in parkwell. It also have a child(ForeignKey) relationship with "Company api" and a parent relationship with "Park Image api".

Park App Fields
Field Uses
company This is a ForeignKey relationship field to the company model.
admin This is a OneToOneField relationship field to the ParkAdminProfile model.
park_name This is the name of the park.
park_email This is the email address of the park.
free_space This take the value number of free parking space of a park. It take only positive integers
duration This is the difference between the arrival and departure time. It take only positive integers.
park_number This is the phone number of the park.
park_address This is the physical location of the park.
park_coordinates This is the coordinate location of the park.
park_about This is the about of the park.
updated_on This is the time the park informations were updated.
created_on This is the time the park informations were created.

  • Park List:

    This display all the parks and their instances (i.e You can access each park and its instance) in a List(python) or Array(JS). And each park in form of Dictionary(Python) or Object(JS).
    NOTE: that this only takes the GET method!!!

    Endpoint: https://www.parkwell.africa/api/v1/park/

  • Park Detail:

    This display only a park and its instance.
    NOTE: that this only takes the GET method for all users!!!

    Endpoint: https://www.parkwell.africa/api/v1/park/{park_id}/
    • "{park_id}" means the park id.

  • Park Create:

    This create a new park and its instance.
    NOTE: that this only takes the POST method for the company admin but takes nothing for unauthorized users!!!

    Endpoint: https://www.parkwell.africa/api/v1/park/create/{user_id}/
    • "{user_id}" means the user id.

  • Park Update:

    This update a park and its instance.
    NOTE: that this only takes the PUT method for the park admin but takes nothing for unauthorized users!!!

    Endpoint: https://www.parkwell.africa/api/v1/park/update/{park_id}/
    • "{park_id}" means the park id.

  • Park Delete:

    This delete a park and its instance.
    NOTE: that this only takes the DELETE method for the park admin but takes nothing for unauthorized users!!!

    Endpoint: https://www.parkwell.africa/api/v1/park/delete/{park_id}/
    • "{park_id}" means the park id.

Account api:

This api handles all processing under the control of the account(registration, login, logout) application in parkwell.

Company App Fields
CompanyAdminProfile Model
Field Uses
admin This is a OneToOneField relationship field to the User model.
company_admin_photo This is the photo of the company's admin(optional).
company_admin_number This is the phone number of the company's admin.
updated_on This is the time the company's admin informations were updated.
created_on This is the time the company's admin informations were created.
ParkAdminProfile Model
Field Uses
admin This is a OneToOneField relationship field to the User model.
park_admin_photo This is the photo of the park's admin(optional).
park_admin_number This is the phone number of the park's admin.
updated_on This is the time the park's admin informations were updated.
created_on This is the time the park's admin informations were created.
User Model
Can update fields
Field Uses
username This is the user's username.
first_name This is the user's first name.
last_name This is the user's last name.
email This is the user's email.
Can not update fields
Field Uses
id This is the user's id.
last_login This is the user's last login info.
is_superuser This return true/false depending on the user's is_superuser.
is_staff This return true/false depending on the user's is_staff.
is_active This return true/false depending on the user's is_active.
date_joined This return datetime the user joined parkwell.

  • User List:

    This display all the users and their instances for all parks in parkwell (i.e You can access each user and its instance) in a List(python) or Array(JS). And each company in form of Dictionary(Python) or Object(JS).
    NOTE: that this only takes the GET method!!!

    Endpoint: https://www.parkwell.africa/api/v1/account/user/

  • User Detail:

    This display only a user and its instance.
    NOTE: that this only takes the GET method for all users!!!

    Endpoint: https://www.parkwell.africa/api/v1/account/user/{token}/
    • "{token}" means the user id.

  • User Update:

    This update a user and its instance.
    NOTE: that this only takes the PUT method for the user admin but takes nothing for unauthorized users!!!

    Endpoint: https://www.parkwell.africa/api/v1/account/user/update/{user_id}/
    • "{user_id}" means the user id.

  • User Delete:

    This delete a user and its instance.
    NOTE: that this only takes the DELETE method for the user admin but takes nothing for unauthorized users!!!

    Endpoint: https://www.parkwell.africa/api/v1/account/user/delete/{user_id}/
    • "{user_id}" means the user id.

  • Register:

    This register a new user instance in to the parkwell database.
    NOTE: that this only takes the POST method!!!

    Endpoint: https://www.parkwell.africa/api/v1/account/register/

  • Register As Company Admin:

    This combine the register model and the company admin model.
    NOTE: that this only takes the POST method!!!
    it takes fields as follow: username, email, password1, password2, and company_admin_number.

    Endpoint: https://www.parkwell.africa/api/v1/account/register_as_company_admin/

  • Login:

    This login in a registered user to the parkwell system with a token set to expire on time duration.
    NOTE: that this only takes the POST method!!!

    Endpoint: https://www.parkwell.africa/api/v1/account/login/

  • Logout:

    This logout a logged in user in parkwell system immediately by calling the delete method on the token.
    NOTE: that this only takes the POST method!!!

    Endpoint: https://www.parkwell.africa/api/v1/account/logout/
